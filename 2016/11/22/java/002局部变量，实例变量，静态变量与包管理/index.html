局部变量，实例变量，静态变量与包管理

发表于 2016-11-22

一、局部变量，实例变量，静态变量

局部变量

局部变量：不是声明在类体括号里面的变量。
（1）必须要先赋值，后使用，否则不能通过编译，局部变量没有默认的初始化值。
（2）作用范围：定义开始到定义它的代码块结束。
（3）同一范围内，不允许2个局部变量命名冲突。

静态变量与实例变量

静态变量和实例变量的区别？

1.语法定义上的区别：静态变量前要加static关键词，而实例变量前则不加。

2.程序运行时的区别：实例变量属于某个对象的属性，必须创建了实例对象，其中的实例变量才会被分配空间，才能使用这个实例变量。
静态变量不属于某个实例对象，而属于类，所以也成为类变量，只要程序加载了类的字节码，不用创建任何实例对象，静态变量就会被分配空间，静态变量就可以使用了。

3.总之，实例变量必须创建对象后才可以通过这个对象被使用，静态变量则可以直接使用类名来使用。

但是，每创建一个实例对象，就会分配一个instanceVar,即可能分配多的个instanceVar，并且每个instanceVar的值都只自加了一次。

类的静态变量在内存中只有一个，java虚拟机在加载类的过程中为静态变量分配内存，静态变量位于方法区中，被类的所有实例共享。
静态变量可以直接通过类名来访问，其生命周期取决于类的生命周期。
而类的实例变量取决于类的实例。每创建一个实例，java虚拟机就会为实例变量分配一次内存，实例变量位于堆栈区中，其生命周期取决于实例的生命周期。

二、包与导入语句解剖

包

包（package）：用于将完成不同功能的类分门别类，放在不同的目录（包）下。包的命名规则：将公司的域名反转作为包名。比如：
www.google.com,则包名字就可以写成com.google（包名），对于包名:每个字母都要小写。

import的语法： import com.google.*

关于package,import,class的顺序问题

1.首先需要定义包（package），可选
2.接下来使用import进行导入，可选
3.然后才是class或interface的定义

注: 如果两个包在同一包下，则不需要导入，直接使用即可。
